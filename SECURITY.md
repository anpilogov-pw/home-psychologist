# Security Policy

## Supported Versions

This project supports security updates for the following versions:

| Version | Supported            |
| ------- | -------------------- |
| 1.0.x   | :white\_check\_mark: |
| < 1.0   | :x:                  |

*This policy applies to the `homepsychologist` WordPress theme version 1.0.0.*

## Dependencies

This theme uses the following key dependencies:

* Tailwind CSS CLI 4.1.6
* @tailwindcss/typography 0.5.16
* Tailwind CSS 4.1.6

Ensure these dependencies are kept up-to-date to mitigate known vulnerabilities.

## Reporting a Vulnerability

If you discover a security vulnerability in this theme, please report it responsibly by following these guidelines:

* **Contact Method:** Send an email describing the issue to \[[your-email@example.com](mailto:your-email@example.com)]

* **Information to Include:**

  * Detailed description of the vulnerability
  * Steps to reproduce
  * Potential impact
  * Any suggested fixes or mitigations

* **Response Time:** You can expect an acknowledgment of your report within 48 hours.

* **Resolution:** We aim to investigate and provide a fix or mitigation within 14 calendar days, depending on the severity.

## What Happens After Reporting

* Reports will be reviewed and verified promptly.
* If confirmed, a security fix will be prepared and released as soon as possible.
* We will notify you of the resolution or request further information if needed.
* If a reported issue is invalid or not a security concern, we will explain why.

## Security Best Practices for Users

* Always keep the theme updated to the latest supported version.
* Regularly update WordPress core and any plugins to their latest secure versions.
* Use trusted hosting and security plugins to enhance site protection.
